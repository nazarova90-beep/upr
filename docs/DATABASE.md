# Database — база данных

Верхнеуровневый обзор хранения данных в UPR. Полная картина выбранного стека и план масштабирования — в `stack.md`.

## Статус

**База данных выбрана:**
- **MVP:** **SQLite** (один файл на диске).
- **ORM:** **SQLAlchemy + SQLModel** — единый словарь моделей для приложения, миграций и будущей админки.
- **Этап 3:** переезд на **PostgreSQL** через ту же ORM (без переписывания приложения).

Подробное обоснование выбора — в `stack.md` → «Почему именно эти варианты».

## Что мы храним

Главные сущности (предварительный список):

- **User** — пользователи (email, хеш пароля, базовый профиль). **Закладка на i18n:** предпочитаемый язык интерфейса (`locale`), когда появится выбор языка — до этого можно хранить значение по умолчанию `ru` или выводить из клиента при необходимости аналитики.
- **Exercise** — упражнения из базы (название, описание техники, мышечные группы).
- **Workout** — тренировки пользователя (список упражнений в порядке).
- **WorkoutSession** — конкретная сессия выполнения тренировки.
- **Set** — подход (вес, повторения).
- **ExerciseChat** — чат по упражнению (один на пару user × exercise).
- **ChatMessage** — сообщения в чате (текст / видео / разметка / системное).
- **VideoAnalysis** — результат разбора видео (метаданные, ссылка на скриншот).
- **Subscription** — подписка (тариф, статус, период).

## Особенности хранения видео

Видео **не хранится в основной БД** — слишком большие файлы. В БД лежит только **ссылка** + метаданные (размер, длительность, тип, владелец, дата истечения).

- **MVP:** ссылка указывает на файл в **локальной папке** (через абстракцию `storage/` в бэкенде).
- **Этап 2+:** та же ссылка указывает на файл в **S3-совместимом хранилище** — без изменения схемы БД.

См. `SECURITY.md` → раздел «Хранение и удаление видео».

## Правила удаления данных

Подробно описаны в:
- `product-specs/exercise-chat.md` — хранение сообщений (2 месяца / бессрочно).
- `SECURITY.md` — удаление при удалении аккаунта.

Ежедневно работает **фоновая задача**, которая удаляет:
- сообщения старше 2 месяцев у пользователей на бесплатном тарифе;
- видео, у которых истёк срок хранения;
- soft-deleted аккаунты, прошедшие grace-period.

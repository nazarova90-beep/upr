# Frontend — мобильное приложение

Верхнеуровневый обзор клиентской части UPR. Полная картина выбранного стека и плана масштабирования — в `stack.md`.

## Статус

**Стек выбран:** **Flutter** (Dart). Один код для iOS и Android, полностью кастомный UI под наш будущий брендбук без привязки к нативным компонентам. Подробное обоснование (почему Flutter, а не React Native) — в `stack.md` → раздел «Почему именно эти варианты».

## Платформы

- **iOS** (минимум поддерживаемой версии — TBD).
- **Android** (минимум поддерживаемой версии — TBD).

## Зона ответственности клиента

- Весь UI приложения.
- Локальное состояние (текущая тренировка, кэш чатов).
- Запись и предварительная валидация видео (длительность, размер, формат).
- Загрузка видео на бэкенд.
- Обновление UI при готовности разбора (через polling, websocket или push — TBD).
- Хранение токена авторизации в безопасном хранилище (Keychain/Keystore).
- Биометрический lock приложения.

## Язык интерфейса и локализация (i18n)

**Продуктовое правило (MVP):** все экраны и системные сообщения клиента — **на русском**. Другие языки в первой публичной версии **не обязательны**.

**Инженерное правило:** закладываем **мультиязычность с первого дня кода**, чтобы не переписывать приложение целиком при добавлении локали.

| Что делаем | Зачем |
|---|---|
| Строки UI вынесены в **ресурсы перевода** (ключ → текст), а не разбросаны литералами по виджетам | Новый язык = новый файл перевода, без охоты по коду |
| Единый **источник истины для форматов** дат, чисел, единиц веса (кг/lb позже через настройки) | Локали по-разному показывают числа и даты |
| Код и комментарии разработчиков — как удобно команде; **продуктовый текст пользователя** — только через слой локализации | Чёткое разделение |
| При выборе стека учесть стандартный для него **механизм i18n** (например, ARB/intl во Flutter, react-i18next и т.д.) | Меньше самописной боли |

**Будущее (после MVP):** язык приложения из настроек или системный язык ОС; синхронизация предпочтения с бэкендом (см. `DATABASE.md`, `BACKEND.md`) для локализованных серверных ошибок и push (когда появятся).

Связанный продуктовый контекст: `product-specs/product.md` → раздел «Язык интерфейса и мультиязычность».

## Библиотеки (предварительный список — пройдёт через `user-context7` перед использованием)

| Зачем | Кандидат |
|---|---|
| Запись и превью видео | `camera`, `video_player` |
| Сетевой клиент к бэкенду | `dio` |
| Управление состоянием | `riverpod` (или `bloc` — выберем при первом коде) |
| Безопасное хранилище токенов | `flutter_secure_storage` |
| Локализация (i18n) | встроенный `intl` + ARB-файлы |
| Pose estimation на устройстве (опционально) | `google_mlkit_pose_detection` или Quickpose |
| Иконки/SVG | `flutter_svg` |

## Что входит в MVP-реализацию

- Один пользователь (без экрана регистрации/логина).
- Список упражнений → экран упражнения → запись/выбор видео → отправка на бэкенд.
- Чат с упражнением: текст + ответ AI + размеченный кадр.
- Тёмная тема, тексты — на русском через ARB-файлы.

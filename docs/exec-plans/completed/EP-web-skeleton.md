# Plan: web/ skeleton — Track D close

Closes Phase 1 / Track D of `roadmap.md` § 5: builds the structural skeleton of `web/` (React + Vite + TypeScript) that makes the Phase 1 exit-trigger fully observable — `web/` skeleton runs locally on Mac Safari, Lucent dark theme rendered, route shells in place. Symmetric to `completed/EP-phase1-track-c-skeleton.md` (which delivered `backend/` + frozen `mobile/`). No business logic — file tree first; thin slice (real Gemini call) is Phase 2 of the roadmap, not this plan.

## Goal & Context

`web/` directory does not exist yet. The active client surface for the project moved from `mobile/` (frozen 2026-04-27 by `EP-pivot-to-web.md`) to a new browser app at `web/`. This plan creates `web/` from scratch: the Vite + React + TypeScript scaffold, the routing shell (`/` Workout + `/chat/:exerciseId` Chat), Lucent CSS wired in, i18next initialised with browser language detection, domain TS types ported from `mobile/src/domain/` (which were authored to mirror backend models), and the empty `api/` + `components/` slots placeholders. Mockups (`docs/ui/mockups/*.html`) and the component inventory (`docs/ui/components.md`) are the visual source of truth the route shells render against; no component is implemented in code yet — slots only.

The plan also closes two Lucent gaps inherited from `completed/EP-track-b-mockups.md` § "What was dropped or deferred": (a) author `.bottom-sheet` in `docs/ui/design-system/style.css` and re-render `docs/ui/mockups/technique-popup.html` against it (replaces the current 340 px centered modal fallback); (b) author a token-driven `.icon-btn` family in `style.css` and replace the four inline-styled ad-hoc circular icon buttons (`.exercise-icon-btn`, `.header-icon-btn`, `.input-icon-btn`, `.modal-close`) across all four mockup HTML files. Both gaps are explicitly assigned to `EP-web-skeleton.md` by the predecessor plan; they live in their own phase (Phase 5) so the actual `web/` scaffold (Phases 2–4) is unblocked.

After this plan closes, Phase 1 of `roadmap.md` reaches its full exit-trigger: Track A specs ✅ (2026-04-19), Track B mockups ✅ (2026-04-27), Track C `backend/` skeleton ✅ (2026-04-27), Track D `web/` skeleton ✅ (this plan). Roadmap then advances to Phase 2 — thin slice (separate plan, started after this one is committed).

### Lessons

1. **Vite alias for the design system pays off immediately.** Single line `@import "@lucent/style.css"` in `tokens.css` plus an alias entry in `vite.config.ts` eliminates the entire "drift between source-of-truth and consumed copy" failure mode that would otherwise reappear on every Lucent edit. Pattern reusable for any future shared-asset folder (e.g., shared icons).
2. **`create-vite` template drift is real and silent.** `create-vite@9` had three meaningful divergences from what the plan expected (TypeScript project-references in `tsconfig.json`, no `vite-env.d.ts`, richer demo asset set). Each was caught at execution time and recorded in Surprises & Discoveries. Lesson: when a plan's literal step says "open `tsconfig.json` and add X", expect the template version to have moved; budget a "verify and adapt" step rather than a "edit blindly" step.
3. **TypeScript 6 + `verbatimModuleSyntax: true` adds small but real friction.** `i18next.use(...).init(...)` returns `Promise<TFunction>`, which strict TS / `no-floating-promises` flags as unhandled. The fix is to wrap in `void i18n.use(...).init(...)`; the side-effect-import contract stays intact. Worth knowing for any future `init`-style chain that returns a promise but is intentionally fire-and-forget.
4. **Owner uses always-on VPN; localhost is the safe default.** The pivot plan's premise (VPN blocks LAN reach to FastAPI from iPhone Safari) is reaffirmed by Phase 6: `localhost:5173` and `localhost:8000` work without any tunnel, on the same Mac. Cloudflared / ngrok are escape hatches when iPhone Safari is needed; not part of the daily loop.
5. **MCP browser is a useful Mac Safari proxy for visual checks.** The cursor-ide-browser MCP cannot fully impersonate Safari (Chromium underneath), but it captured: dark Lucent background, Manrope rendered, Russian strings, parametrized URL, clean console — all the literal items the Phase 6 done-when listed. Owner can still re-open in real Mac Safari any time; no contradiction expected.
6. **Splitting Lucent gaps into their own phase was the right call.** Phase 5 (`.bottom-sheet` + `.icon-btn` cleanup) ran independently from the Vite scaffold (Phases 2–4) and required no `web/` context — exactly what the plan's phase-sizing rationale predicted. A phase that only touches `docs/ui/design-system/style.css` and `docs/ui/mockups/*.html` fits comfortably in a single fresh-chat agent run.
